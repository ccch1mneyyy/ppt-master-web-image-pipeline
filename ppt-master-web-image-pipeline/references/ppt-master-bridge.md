# PPT Master Bridge

This skill is a narrow bridge into `ppt-master` Step 5 Image_Generator. It should leave the rest of the PPT Master workflow untouched.

## Input Contract

Read these inputs from the project before generating anything:

- `design_spec.md`
- Section I project metadata
- Section III visual theme and color scheme
- Section VIII Image Resource List

The Image Resource List is the source of truth for:

- filename
- requested dimensions
- image type
- current status
- generation description

Never rename files or invent alternate storage locations. Every generated asset must land in `project/images/<filename>`.

## Output Contract

This skill must produce:

- `project/images/image_prompts.md`
- `project/images/web_generation_manifest.json`
- generated image files in `project/images/`
- updated `design_spec.md` statuses for completed rows

If an item cannot be cleanly generated by a website, leave the row unresolved and explicitly call it out in the handoff note.

## Handoff Rules

After finishing this skill, Executor should be able to continue without guessing:

- `image_prompts.md` documents what was attempted
- the generated files already exist at final filenames
- pending rows reflect real blockers, not forgotten work

Do not modify:

- `svg_output/`
- `svg_final/`
- speaker notes
- page layout decisions outside the image rows

## Type-Specific Guidance

- `Background`: good fit for website image models. Preserve negative space for slide titles.
- `Photography`: good fit for website image models. Prefer believable lighting and one clear focal subject.
- `Illustration`: good fit for website image models. Keep palette aligned with the deck.
- `Decorative`: good fit for website image models. Keep it subtle.
- `Diagram`: use caution. If the slide needs exact labels, precise arrows, or data fidelity, route the work back to PPT Master SVG construction instead of forcing a brittle web-generated image.

## Sequence Discipline

Follow the same practical discipline as PPT Master Step 5:

1. extract image rows
2. build prompt document
3. generate one image at a time
4. save under exact filename
5. clean any watermark or edge issue
6. verify
7. update status
8. hand back to Executor
